Ajout de données structurées sur les pages de Canada.ca - Instructions pour le projet pilote

De : Secrétariat du Conseil du Trésor

En mars 2020, le Bureau de la transformation numérique a lancé un projet pilote sur les données structurées. De nombreux ministères y participent.

Ces instructions sont une ébauche. Elles ne sont rédigées que pour aider à ajouter des données structurées aux pages de Canada.ca pour ce projet pilote.

Il ne s'agit pas de directives officielles pour Canada.ca.

Nous découvrons comment les données structurées aident les gens à trouver des réponses et à accomplir leurs tâches.

L'objectif de ce projet pilote est de découvrir les obstacles techniques et les problèmes liés à l'organisation du travail. Il vise également à découvrir les données structurées les plus efficaces pour différents contextes.

Que sont les données structurées

Les données structurées sont un moyen d'étiqueter le contenu de manière à ce qu'il soit lisible par une machine.

Elles vous permettent d'ajouter une signification à la page.

Cela devrait aider à créer:

  • des résultats de recherche meilleurs et plus informatifs (souvent dans Google et Bing)
  • de meilleurs résultats de recherche vocale (pour des services comme Alexa et Google Home)

Notez que vous ne pouvez pas contrôler la façon dont les résultats de recherche apparaissent (par exemple, ce qui apparaît dans l'extrait en vedette). L'ajout de données structurées peut modifier ou non les résultats de la recherche.

Les données structurées doivent être une "représentation fidèle" de la page. Vous ne pouvez pas ajouter de données structurées pour un contenu qui n'est pas abordé dans la page.

L'ajout de données structurées n'est qu'une des techniques que vous pouvez utiliser pour optimiser votre contenu pour la recherche. La première étape consiste à améliorer le contenu réel de la page.

  • Google : Comprendre le fonctionnement des données structurées
  • Google : Suivre les consignes relatives aux données structurées
  • Optimiser votre contenu pour la recherche vocale

Comment ajouter des données structurées

Étape 1 : Déterminer le schéma le plus efficace à ajouter

Il existe de nombreux schémas de données structurées que vous pouvez utiliser pour ajouter une signification à votre page.

Consultez la liste des schémas sur Schema.org ou Google, ou regardez les exemples fournis sur cette page.

  • Schémas Schema.org (en anglais seulement)
  • Données structurées Google
  • Code et exemples de schémas pour Canada.ca

Étape 2 : Choisissez la méthode

Vous pouvez utiliser 2 méthodes différentes pour ajouter des données structurées à une page de Canada.ca : JSON-LD et RDFa. Vous pouvez également utiliser une combinaison des deux méthodes.

La documentation de chaque schéma fournit généralement le balisage pour chaque méthode.

Ajouter un script JSON-LD

  • le script est ajouté sous la forme d'extrait de code séparé du contenu de la page
  • un exemple de code peut être copié/collé et les valeurs ajustées
  • nécessite une duplication du contenu
  • risque de conflit avec les données structurées déjà intégrées dans les gabarits de la BOEW
  • méthode recommandée par Google

Ajouter du code RDFa

  • les attributs sont ajoutés aux éléments individuels de la page
  • peut nécessiter l'ajout d'éléments supplémentaires au code afin de créer une structure, tels que section, div et span.
  • pas de duplication de contenu
  • risque de conflit avec les données structurées déjà intégrées dans les gabarits de la BOEW

Étape 3 : Tester le code de données structurées

Vous pouvez copier des extraits de code ou des URL complètes pour tester le code avec différents outils :

  • Tests de résultats de recherche riches (en anglais seulement) : montre quels types de résultats riches (supportés par Google) ont été trouvés sur la page, ainsi que toute erreur ou suggestion concernant vos données structurées.
  • Test de données structurées (en anglais seulement) : valide les données structurées.

Les deux outils devraient donner le même résultat. En collant le code, vous pouvez voir s'il est valide et si les valeurs attribuées aux différents champs sont correctes.

Étape 4 : Valider le code HTML et vérifier l'accessibilité

Avant de publier la page avec des données structurées, assurez-vous que le HTML est valide et que la page est accessible. Vous pouvez utiliser des validateurs en ligne pour vous aider à cet effet :

  • HTML Validateur HTML W3C (en anglais seulement)
  • WAVE Web Accessibility Evaluation Tool (en anglais seulement)

Étape 5 : Publier la page, réindexer et suivre les résultats

Une fois vos données structurées publiées, demandez à Google et à Bing de réindexer la page.

Vérifiez l'état de votre code dans la console de recherche Google de Canada.ca.

Code et exemples de schémas

SpecialAnnouncement

Quand l'utiliser

Le schéma specialAnnouncement est un nouveau schéma expérimental développé pour les annonces liées à la COVID-19. Il est destiné principalement aux informations liées à la santé (symptômes, quarantaine, tests, etc.), mais aussi aux fermetures, aux réouvertures et aux prestations spéciales du gouvernement.

Directives officielles

  • Directives de Schema.org pour specialAnnouncement (en anglais seulement)
  • Directives de Google pour specialAnnouncement

FAQPage

Quand l'utiliser

Permet d'ajouter un ensemble de "questions et réponses" (ou un ensemble de rubriques et leur contenu respectif). Vous pouvez soit baliser le contenu visible (en utilisant RDFa), soit ajouter une couche en JSON-LD.

À utiliser sur les pages suivantes :

  • Page de lancement de service
  • Documents de directive
  • Foires aux questions (FAQ), bien que les pages FAQ soient fortement déconseillées sur Canada.ca dans la plupart des cas - lisez Éviter les FAQ, à une exception près
